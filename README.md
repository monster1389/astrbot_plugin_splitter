## 简介
对话分段PRO，优化对话分段的体验，为小白提供更好的引导（不懂正则也能玩！）。  
将长文本智能切割为多段短消息，并配合可调的延迟算法，模拟真人的输入频率与节奏。

插件提供三种配置模式，从开箱即用到深度自定义，满足不同用户的需求。

## 配置模式

| 模式 | 说明 | 适合人群 |
| :--- | :--- | :--- |
| **简易模式** | 只暴露几个核心开关，无需理解技术概念即可使用。内置最优默认值，开箱即用。 | 新手、不想折腾的用户 |
| **进阶模式** | 提供列表式配置界面，可自定义分段符号、清理文本、替换规则等。不需要写正则，填写文本即可。 | 有一定需求、想微调行为的用户 |
| **专业模式** | 开放全部配置项，使用正则表达式实现精细控制。包括分段正则、前后置清理正则、延迟参数、组件策略等。 | 高级用户、需要高度自定义的场景 |

三种模式可随时切换，切换后保存并重载插件即可生效。

## 功能介绍

### 分段识别
支持简单符号列表与高级正则表达式两种分段方式。简单模式下只需填写符号（如 `。` `！` `?`），遇到这些符号即自动断开；专业模式下可使用完整正则表达式精确控制断点位置。

### 智能保护
- **成对符号保护**：自动识别括号、引号、书名号等成对符号，确保其内部内容不被切断。
- **代码块保护**：识别 Markdown 代码块（\`\`\`），将整段代码作为整体保留。
- **思维链保护**：识别 `<think>...</think>` 标签，避免推理过程被意外拆分。
- **保护词机制**：可配置特定文本，当分段点紧邻这些文本时不会断开。例如填入 `Provider`，则「测试。 Provider 是...」不会在句号后断开。

### Markdown 表格保护
自动检测以 `|` 开头的连续行，将整个 Markdown 表格识别为一个整体单元，避免表格被分段导致格式错乱。包括表头分隔行（`|---|---|`）也在保护范围内。

### 中英文混排优化
针对中英文混合场景做了专门处理：
- 纯英文上下文中，避免在句点、逗号等标点处过度分段，保持英文句子的完整性。
- 中文与英文/数字相邻时，不会在同位置产生多余的断行，输出更加自然流畅。

### 文本清洗与替换
- **前置清洗**：在分段处理前先行删除指定文本（如思维链标记、冗余提示语等），减少无效分段。
- **后置清洗**：分段完成后逐段清理残留内容。
- **批量替换**：支持配置多条查找替换规则，分段前统一执行。所有规则同时应用，避免顺序执行导致的交叉覆盖问题。支持 `\n`（换行）`\t`（制表符）`\s`（空格）转义。

### 反向替换
开启后，替换规则会双向生效：
- AI 回复中的「原始文本」会被替换为你指定的「目标文本」。
- 用户发送的「目标文本」会被还原为「原始文本」后再交给 AI。

例如配置 `1 -> 2`，则 AI 输出的 `1` 会显示为 `2`，用户输入的 `2` 会被还原为 `1` 再交给模型。此功能也可用于规避部分模型的安全检查机制。

### 拟真延迟
内置四种延迟策略，根据下一段文字长度自动计算发送间隔：
- **线性**：延迟 = 基础值 + 字数 x 系数，字数越多等越久。
- **对数**：延迟随字数增长但增速递减，适合大段文字。
- **随机**：在设定范围内随机取值，模拟不可预测的阅读节奏。
- **固定**：每段间隔固定值，简单直接。

### 组件控制
针对图片、@ 提及、表情等非文本组件，可分别设定独立的发送策略：
- **单独**：组件独立一条消息发出。
- **跟随下段**：组件与下一段文字合并。
- **跟随上段**：组件与上一段文字合并。
- **嵌入**：组件保留在原始位置上不移动。

### 智能均分
根据总字数与分段上限，自动计算理想段长，尽量保持每条消息篇幅均衡。可调节均分上下限比例，避免出现过长或过短的碎片消息。尾部过短的段落会自动合并到前一段。

### TTS 语音适配
支持将每段文字转换为语音消息。开启后，插件会检查当前对话的 TTS 配置，根据触发概率逐段生成语音。支持双输出模式：同时发送语音和文本消息，兼顾听和看的需求。

### 智能回复与防重复
- **智能回复**：发送前检测当前消息之后是否已有新消息插入，有则给第一段自动附加 Reply 引用，避免上下文错乱。
- **并发锁保护**：同一对话的分段处理使用异步锁，防止多个分段任务同时执行导致消息重复发送。

### 多端适配
- 群聊分段可独立开启或关闭。
- 支持对话级黑白名单，精确控制分段生效范围。
- 分段范围可选择「仅 AI 回复」或「全部输出」，灵活应对不同场景。
- 在钉钉等受限平台自动退避智能回复功能，确保兼容性。
